# 闪避系统 (Dodge System)

> **Status**: Approved
> **Author**: user + agents
> **Last Updated**: 2026-04-24
> **Implements Pillar**: 操作感即玩法、每一秒都在做选择

## Overview

闪避系统管理玩家的快速位移闪避行为，是操作核心和生存手段。玩家按下闪避键（空格/B 键）后，角色沿指定方向快速冲刺一段固定距离——移动中朝移动方向闪避，静止时朝鼠标瞄准方向闪避。闪避位移期间拥有无敌帧，免疫所有伤害，但被墙壁阻挡。闪避期间可以攻击，允许高风险的闪避反击操作。闪避有冷却时间（防止无限闪避），冷却中按闪避键无效。闪避系统从输入系统接收 `dodge` 触发信号，闪避期间覆盖移动系统的位移控制，闪避结束后归还控制权。闪避触发时设置生命/伤害系统的 `is_invincible` 标志，无敌帧时长不与受击无敌帧叠加，取较长者。

## Player Fantasy

你被三只敌人包围，一只从正面扑来。你按下空格——角色朝侧方飞速滑出，敌人的攻击从你虚影中穿过。你还在位移途中就按下了左键，大剑在滑行中挥出，砍中追来的敌人。那一瞬间，你同时做了两件事：**躲开了死亡，反击了敌人**。这就是闪避反击——高风险高回报的操作。但闪避不是无限的：冷却计时让你必须精打细算每一次闪避的时机。用早了，下一波攻击你无处可躲；用晚了，你已经中招。每一次成功的闪避都让你觉得自己是读心者——你预判了敌人的攻击，在最后一刻做出了正确的选择。

## Detailed Design

### Core Rules

1. **闪避触发**
   - 从输入系统接收 `dodge` 的 `just_pressed` 信号
   - 冷却中（`dodge_cooldown` > 0）按闪避键无效
   - 玩家处于 DYING/DEAD 状态时无法闪避
   - 每次按下只触发一次闪避，按住空格不连续闪避

2. **闪避方向**
   - 如果玩家正在移动（移动系统的 `direction` 非零）：闪避方向 = 移动方向
   - 如果玩家静止（`direction` 为零）：闪避方向 = 战斗系统的 `aim_direction`（鼠标方向）
   - 闪避方向在位移开始时锁定，中途不改变

3. **闪避位移**
   - 固定冲刺距离：`dodge_distance`（默认 120px）
   - 固定冲刺速度：`dodge_speed`（默认 600 px/s）
   - 闪避持续时间 = `dodge_distance / dodge_speed`（默认 0.2s）
   - 使用 `move_and_slide()` 处理碰撞，被墙壁阻挡时沿墙壁滑动

4. **无敌帧**
   - 闪避触发时设置 `is_invincible = true`，持续 `dodge_invincibility_duration`（默认 0.2s，与闪避持续时间相同）
   - 无敌帧与受击无敌帧（0.2s）不叠加，取剩余时间较长者
   - 无敌帧结束后 `is_invincible = false`

5. **闪避期间攻击**
   - 闪避期间可以正常触发近战和远程攻击
   - 攻击方向使用战斗系统的 `aim_direction`（鼠标方向），不受闪避位移方向影响
   - 闪避期间的攻击冷却、伤害计算与正常状态完全一致

6. **冷却**
   - 闪避结束后进入冷却：`dodge_cooldown`（默认 2.0s）
   - 冷却期间无法再次闪避
   - 冷却可被升级系统修改：`dodge_cooldown_down` 升级每层 -0.3s（最低 0.3s）
   - 闪避触发时发出 `dodge_started()` 信号，供视觉反馈系统生成残影

7. **与移动系统的交互**
   - 闪避期间：移动系统停止响应 WASD 输入，位移由闪避系统控制
   - 闪避期间 WASD 输入不影响闪避方向（方向在触发时锁定）
   - 闪避结束后：立即恢复移动系统控制，WASD 输入正常响应

### States and Transitions

| 状态 | 进入条件 | 退出条件 | 行为 |
|-------|---------|---------|------|
| `READY` | 冷却结束 / 初始状态 | 收到 `dodge` just_pressed | 可以闪避，等待触发 |
| `DODGING` | 收到 `dodge` just_pressed 且冷却结束 | 闪避位移完成（时间到） | 沿锁定方向快速位移，无敌，可攻击 |
| `COOLDOWN` | 闪避位移完成 | 冷却计时结束 | 无法闪避，其他行为正常 |

### Interactions with Other Systems

| 交互系统 | 方向 | 接口说明 |
|---------|------|---------|
| 输入系统 | 输入 → 本系统 | 提供 `dodge` 的 `just_pressed` 信号 |
| 玩家移动系统 | 本系统 → 移动 | 闪避期间覆盖位移控制（设置 `is_dodging = true`），结束后恢复 |
| 生命/伤害系统 | 本系统 → 生命/伤害 | 闪避触发时设置 `is_invincible = true`，无敌结束时恢复 `false` |
| 混合战斗系统 | 本系统 ↔ 战斗 | 闪避期间可攻击；读取战斗系统的 `aim_direction` 用于静止时闪避方向；闪避状态不影响攻击逻辑 |
| 局内升级系统 | 升级 → 本系统 | 修改 `dodge_cooldown`、`dodge_distance`、`dodge_invincibility_duration` 等参数 |

## Formulas

### 闪避持续时间

```
dodge_duration = dodge_distance / dodge_speed
```

### 闪避位移（每帧）

```
dodge_displacement = dodge_direction * dodge_speed * delta
```

### 冷却倒计时

```
dodge_cooldown_remaining -= delta  # 每帧递减
if dodge_cooldown_remaining <= 0:
    state = READY
```

### 变量表

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `dodge_direction` | Vector2 | 长度 1 | 移动方向或鼠标方向 | 闪避位移方向，触发时锁定 |
| `dodge_distance` | float | 50-250 px | 调参旋钮 | 固定冲刺距离 |
| `dodge_speed` | float | 300-1200 px/s | 调参旋钮 | 固定冲刺速度 |
| `dodge_duration` | float | 0.05-0.5s | 计算 | 闪避持续时间 |
| `dodge_invincibility_duration` | float | 0.05-0.5s | 调参旋钮 | 无敌帧时长 |
| `dodge_cooldown` | float | 0.3-5.0s | 调参旋钮 | 闪避冷却时间 |
| `dodge_cooldown_remaining` | float | 0 到 dodge_cooldown | 运行时 | 冷却剩余时间 |

## Edge Cases

| 场景 | 预期行为 | 理由 |
|------|---------|------|
| 闪避途中撞墙 | 位移停止，沿墙壁滑动，闪避时间正常流逝，无敌帧照常生效 | `move_and_slide()` 自动处理碰撞 |
| 闪避途中撞墙导致实际位移 < `dodge_distance` | 闪避仍在 `dodge_duration` 结束后才进入冷却，不提前结束 | 闪避持续时间固定，不因碰撞缩短 |
| 站在墙角闪避，两个方向都是墙 | 位移为零（卡在原地），但无敌帧仍然生效 | 无敌帧是闪避的保障，不因位置取消 |
| 冷却期间按闪避键 | 忽略，无反馈（不播放音效/动画） | 防止无效操作误导玩家 |
| DYING/DEAD 状态按闪避键 | 忽略 | 死亡后不可闪避 |
| 闪避期间无敌帧与受击无敌帧重叠 | 取剩余时间较长者，不叠加 | 遵守生命/伤害系统的无敌帧规则 |
| 闪避期间同时按攻击键 | 正常执行攻击，攻击方向 = 鼠标方向 | 闪避反击是核心操作 |
| 闪避方向锁定后，移动方向改变 | 闪避方向不变（锁定） | 闪避是"决策"——选择方向的瞬间就是决策时刻 |
| 极短时间内连续按两次闪避键 | 第一次正常闪避，第二次因冷却被忽略 | `just_pressed` 是单帧事件 + 冷却保护 |
| 升级将 `dodge_cooldown` 降为 0 | 仍需至少 0.1s 冷却（断言保护） | 0 冷却会导致无限闪避，破坏游戏节奏 |
| 闪避距离被升级修改后，持续时间自动变化 | `dodge_duration` 随 `dodge_distance / dodge_speed` 重新计算 | 公式驱动，数据修改即时生效 |
| 闪避期间玩家穿过敌人 | 允许，玩家碰撞体不与敌人碰撞体交互 | 遵守移动系统的"玩家不与敌人碰撞"规则 |

## Dependencies

| 系统 | 方向 | 性质 |
|------|------|------|
| 输入系统 | 本系统依赖输入 | 硬依赖：需要 `dodge` 的 `just_pressed` 信号 |
| 玩家移动系统 | 本系统 ↔ 移动 | 硬依赖：闪避期间覆盖位移控制，读取 `direction` 判断闪避方向 |
| 混合战斗系统 | 本系统依赖战斗 | 硬依赖：静止时读取 `aim_direction` 作为闪避方向；闪避期间攻击方向依赖 `aim_direction` |
| 生命/伤害系统 | 本系统 → 生命/伤害 | 硬依赖：设置 `is_invincible` 标志，查询无敌帧剩余时间 |
| 升级池/数据系统 | 升级 → 本系统 | 软依赖：监听 `upgrade_acquired` 信号，`dodge_cooldown_down` 每层 -0.3s，最低 0.3s |
| 视觉反馈系统 | 本系统 → 视觉 | 软依赖：发出 `dodge_started` 信号触发残影特效 |
| 战斗 HUD | 本系统 → HUD | 软依赖：提供 `dodge_cooldown` 和充能状态给 HUD 显示 |

**信号清单（本系统发出）**：

| 信号 | 参数 | 消费者 |
|------|------|--------|
| `dodge_started` | （无参数） | 视觉反馈系统 |

## Tuning Knobs

| 参数 | 当前值 | 安全范围 | 调高效果 | 调低效果 |
|------|--------|----------|---------|---------|
| `dodge_distance` | 120 px | 50-250 | 闪避更远，逃生能力更强 | 闪避更短，需要更精确走位 |
| `dodge_speed` | 600 px/s | 300-1200 | 闪避更快，手感更干脆 | 闪避更慢，容易被击中 |
| `dodge_invincibility_duration` | 0.2s | 0.05-0.5s | 无敌时间更长，更宽容 | 无敌时间更短，需要更精准时机 |
| `dodge_cooldown` | 2.0s | 0.3-5.0s | 闪避更频繁，操作空间更大（下限 0.1s 保底） | 闪避更稀少，每次使用更关键 |

> **注意**：`dodge_duration` 是计算值（`dodge_distance / dodge_speed`），不作为独立旋钮。调整距离或速度会自动改变闪避持续时间。

## Visual/Audio Requirements

| 事件 | 视觉反馈 | 音频反馈 | 说明 |
|------|---------|---------|------|
| 闪避启动 | 角色残影拖尾（2-3 帧半透明残影） | 闪避音效（短促破风声） | 残影沿闪避路径生成 |
| 闪避中（无敌帧） | 角色半透明闪烁 | 无 | 无敌帧视觉提示 |
| 闪避结束 | 残影消散 | 着地音效（轻微） | 标记位移结束 |
| 冷却中按闪避键 | 无 | 无 | 不给无效操作反馈，防止误导 |

## Open Questions

| 问题 | 状态 | 备注 |
|------|------|------|
| 闪避无敌帧时长是否应该与闪避持续时间解耦？ | 待定 | 当前默认相同（0.2s），但升级可能独立修改其中一个，需要验证是否会导致无敌帧长于闪避时间 |
| 闪避是否需要"闪避次数"上限（如最多连续 2 次闪避）？ | 待定 | 当前设计冷却 2.0s 已限制频率，但"闪避次数"可能更适合作为升级维度 |
| 闪避残影粒子是否由视觉反馈系统负责？ | 待定 | 概念上应属于视觉反馈系统，但 MVP 阶段可先在闪避系统内简单实现 |

# 混合战斗系统 (Mixed Combat System)

> **Status**: Approved
> **Author**: user + agents
> **Last Updated**: 2026-04-23
> **Implements Pillar**: 操作感即玩法、每一秒都在做选择

## Overview

混合战斗系统管理玩家的所有攻击行为，是游戏核心体验的枢纽。它接收输入系统的近战和远程攻击信号，结合鼠标位置确定攻击方向，向生命/伤害系统调用 `take_damage()` 接口对敌人造成伤害。玩家装备两把武器——左手武器槽和右手武器槽，每把可以是近战或远程。左键近战、右键远程：当双持同类型武器时，一次按键触发两把武器依次攻击。双近战生成更大的扇形碰撞区域，两把扇形在鼠标方向上部分重叠，重叠区域伤害加成；双远程完成一轮射击（总耗时低于单把连射两次），但附带弹道散射，远距离可能空枪。一近一远是最均衡的配置，双持同类型是极端路线——更高收益配更高风险。战斗系统同时控制人物朝向（始终面朝鼠标方向），覆盖移动系统的默认朝向规则。它读取移动系统的位置数据，接收输入系统的持续攻击信号，是连接"操作"与"伤害结果"的核心中枢。

## Player Fantasy

你朝右移动，鼠标却指向左后方——三个敌人正追来。你按下左键，两把武器依次挥出：先是一把大剑横扫 120°，紧接着一把短刀再补 120°——两把扇形在你面前交叠，重叠区域的敌人吃到更高伤害，三只同时被命中，伤害数字弹起，敌人闪烁后退。你不停歇，继续向右跑，鼠标切到前方按下右键，一发子弹精准命中最前面的那个。你始终在动，但攻击方向完全独立于移动方向——你可以在逃跑时回头挥砍，在冲锋时侧向射击。近战贴脸高风险高回报，远程拉开距离但弹药有限。每一次命中都带来清晰的反馈：伤害数字、敌人闪烁、音效震动——让你觉得每一刀、每一枪都有分量。这个系统让你在混乱中保持优雅：你不是在"按按钮"，你是在指挥一场战斗。

## Detailed Design

### Core Rules

1. **武器槽系统**
   - 玩家拥有两个武器槽：**左手槽** 和 **右手槽**
   - 每把武器有类型（`melee` 或 `ranged`），由武器系统提供数据
   - MVP 默认装备：左手近战（大剑）、右手远程（手枪）
   - 允许配置：1 近 1 远（均衡）、双近战、双远程

2. **鼠标方向读取**
   - 每帧从 Godot 鼠标 API 读取鼠标屏幕坐标
   - 转换为世界坐标方向：`aim_direction = (mouse_world_position - player_position).normalized()`
   - `aim_direction` 为玩家攻击方向，人物始终面朝此方向

3. **朝向控制**
   - 战斗系统每帧将玩家节点旋转至 `aim_direction`（覆盖移动系统的"朝向 = 移动方向"规则）
   - 旋转是即时的，无过渡动画
   - 无鼠标移动输入时保持最后一次 `aim_direction`

4. **攻击触发流程**
   - **左键近战**：触发所有左手槽和右手槽中类型为 `melee` 的武器
   - **右键远程**：触发所有左手槽和右手槽中类型为 `ranged` 的武器
   - 每把武器独立冷却，冷却中不可触发

5. **近战命中判定**
   - 近战攻击在玩家前方（`aim_direction` 方向）生成扇形碰撞区域
   - 扇形参数由武器数据决定（MVP 默认：角度 120°、半径 80px，原型验证 60px 过短）
   - **单近战**：生成一把武器的扇形，命中区域内所有敌人
   - **双近战**：两把武器依次触发（间隔 0.1s），各自生成扇形：
     - 两把扇形中心方向相同（都沿 `aim_direction`），但角度可能不同
     - 两扇形的**重叠区域**内敌人受到 `overlapping_damage`（两把武器伤害相加 × 1.2 倍）
     - 非重叠区域内敌人受到各自武器的正常伤害
     - 双近战时总攻击扇形更大，但非重叠区域伤害不加成

6. **远程弹道**
   - 远程攻击从玩家位置沿 `aim_direction` 发射飞行弹
   - 飞行弹为 `Area2D` 节点，直线飞行，速度 600 px/s
   - 弹体生成时记录 `source = player`（伤害来源引用），命中时传给 `take_damage()`
   - 命中敌人时调用 `take_damage()`，销毁弹体
   - 超出最大射程后自动销毁（MVP 默认 400px）
   - **单远程**：笔直线弹道，无散射
   - **双远程**：单击右键触发一轮射击（两把一起发射）：
     - 一轮射击冷却 = 单把冷却 × 1.5（比两次单独射击快 25%）
     - 每把武器各消耗 1 发弹药，一轮共消耗 2 发
     - 每发弹道在 `aim_direction` 基础上加随机散射偏移 ±5°
     - 散射偏移在近距离影响小，远距离可能导致空枪
     - 两发弹体独立飞行、独立命中、独立伤害
     - 双远程配置下，每把武器造成基础伤害的 90%

7. **攻击冷却**
   - 每把武器独立 `attack_cooldown`（秒），由武器数据提供
   - MVP 默认：近战 0.5s，远程 0.3s
   - 冷却中按攻击键无效——必须等待冷却完成
   - 双持同类型时：
     - 双近战：两把独立冷却，单击左键触发所有已冷却的近战武器
     - 双远程：两把共享一轮冷却（冷却 = 单把 × 1.5），单击右键两把同时发射

8. **近战/远程互斥**
   - 同时按住左键和右键时，只执行先按下的攻击类型
   - 近战攻击动画（0.2s）期间忽略远程输入——近战结束后若远程信号仍有效（按键仍按住）则立即射击
   - 远程攻击为瞬态，不影响近战输入——射击完成后可立即近战

9. **弹药管理**
   - 每把远程武器独立管理弹药（非全局共享）
   - 每把远程武器有弹药上限（MVP 默认 10 发）
   - 每次射击消耗该武器 1 发弹药，弹药为 0 时该武器无法射击
   - 弹药自动回复：该武器不射击时每秒回复 1 发，上限 10 发
   - 双远程配置下，两把武器独立回复：一把射击不影响另一把的弹药和回复计时器

### States and Transitions

| State | Entry Condition | Exit Condition | Behavior |
|-------|----------------|----------------|----------|
| `IDLE` | 无攻击输入 | 按下近战或远程键 | 等待攻击输入，持续更新 aim_direction |
| `MELEE_ATTACKING` | 近战攻击触发 | 近战攻击动画完成（0.2s） | 生成近战扇形碰撞区域，检测命中，调用伤害 |
| `RANGED_ATTACKING` | 远程攻击触发 | 弹体发射完成（瞬间） | 生成飞行弹体，消耗弹药，进入冷却 |
| `COOLDOWN` | 攻击完成后 | 冷却计时器归零 | 对应武器不可用，其他武器可独立攻击 |

- 攻击状态转换是即时的（无过渡动画）
- `MELEE_ATTACKING` 持续时间固定 0.2s（命中判定窗口）
- `RANGED_ATTACKING` 为瞬态（弹体生成后立即进入 `COOLDOWN`）
- `COOLDOWN` 为每把武器独立状态，一把冷却不影响另一把
- 双近战依次触发时：第一把 `MELEE_ATTACKING` → 0.1s 后第二把 `MELEE_ATTACKING`
- 双远程同时触发：两把同时 `RANGED_ATTACKING` → 同时进入 `COOLDOWN`（共享一轮）
- 玩家死亡（DYING/DEAD 状态）下战斗系统完全停止，不再响应输入

### Interactions with Other Systems

| 交互系统 | 方向 | 接口说明 |
|---------|------|---------|
| 输入系统 | 输入 → 本系统 | 提供 `melee_attack` 和 `ranged_attack` 的 is_pressed 信号（持续）；提供鼠标屏幕坐标 |
| 移动系统 | 本系统 ↔ 移动 | 读取玩家 `position`；**覆盖** `facing_direction`（朝向 = aim_direction） |
| 生命/伤害系统 | 本系统 → 生命/伤害 | 调用 `take_damage(amount, damage_type, source)` 对敌人造成伤害 |
| 武器系统 | 武器 → 本系统 | 提供武器数据（伤害值、冷却时间、扇形角度/半径、弹道速度/射程/散射、弹药上限） |
| 视觉反馈系统 | 本系统 → 视觉 | 命中时发送信号，触发伤害数字飘字、敌人闪烁、弹体命中粒子 |
| 音效系统 | 本系统 → 音效 | 近战挥砍、远程射击、命中音效的触发信号 |
| 分数/统计系统 | 本系统 → 统计 | 击杀敌人时发送击杀类型（近战/远程）、伤害值 |

## Formulas

### 鼠标瞄准方向

```
mouse_world = get_viewport().get_camera_2d().get_global_mouse_position()
aim_direction = (mouse_world - player_position).normalized()
```

### 近战扇形碰撞检测

```
# 判断敌人是否在扇形内
to_enemy = (enemy_position - player_position)
distance = to_enemy.length()
angle = aim_direction.angle_to(to_enemy)

if distance <= weapon.melee_radius and abs(angle) <= deg_to_rad(weapon.melee_angle / 2.0):
    # 敌人在扇形内，命中
    damage_to_enemy = weapon.base_damage
```

### 双近战重叠区域伤害

```
# 两把近战武器的扇形分别为扇形A和扇形B
# 重叠区域 = 两扇形角度交集

# 敌人位置判断
to_enemy = (enemy_position - player_position)
distance = to_enemy.length()
angle = aim_direction.angle_to(to_enemy)

# 判断是否在各扇形内（考虑不同半径）
in_a = distance <= weapon_a.melee_radius and abs(angle) <= deg_to_rad(weapon_a.melee_angle / 2.0)
in_b = distance <= weapon_b.melee_radius and abs(angle) <= deg_to_rad(weapon_b.melee_angle / 2.0)

# 重叠区域 = 同时在两个扇形内
if in_a and in_b:
    overlapping_damage = (weapon_a.base_damage + weapon_b.base_damage) * 1.2
elif in_a:
    damage = weapon_a.base_damage
elif in_b:
    damage = weapon_b.base_damage
```

### 远程弹道（含散射）

```
# 单远程：笔直弹道
bullet_direction = aim_direction
bullet_velocity = bullet_direction * weapon.bullet_speed

# 双远程：带散射
scatter_angle = randf_range(-weapon.scatter_degrees, weapon.scatter_degrees)
bullet_direction = aim_direction.rotated(deg_to_rad(scatter_angle))
bullet_velocity = bullet_direction * weapon.bullet_speed

# 弹体每帧移动
bullet.position += bullet_velocity * delta
```

### 弹药回复（每把武器独立）

```
# 每把远程武器独立管理弹药和回复计时器
# 该武器不射击时每秒回复 1 发
if time_since_last_shot >= 1.0 and ammo < max_ammo:
    ammo += 1
    time_since_last_shot = 0.0
```

### 双远程伤害

```
# 双远程配置下，每把武器造成基础伤害的 90%
dual_ranged_damage = floor(weapon.base_damage * 0.9)
# 例：base_damage=15 → dual_ranged_damage=13
# 一轮射击总伤害 = 13 + 13 = 26（vs 单远程 15）
# DPS 对比：单远程 15/0.3s = 50 DPS；双远程 26/0.45s ≈ 58 DPS（+16%）
```

### 变量表

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `aim_direction` | Vector2 | 单位向量 | 计算 | 鼠标瞄准方向 |
| `weapon.base_damage` | int | 1-999 | 武器系统 | 武器基础伤害 |
| `weapon.melee_angle` | float | 60-180° | 武器系统 | 近战扇形角度 |
| `weapon.melee_radius` | float | 30-100px | 武器系统 | 近战扇形半径 |
| `weapon.bullet_speed` | float | 300-1000 px/s | 武器系统 | 弹体飞行速度 |
| `weapon.scatter_degrees` | float | 0-15° | 武器系统 | 弹道散射角度 |
| `weapon.attack_cooldown` | float | 0.1-2.0s | 武器系统 | 攻击冷却时间 |
| `ammo` | int | 0-max_ammo | 每把武器独立 | 该武器当前弹药量 |
| `max_ammo` | int | 5-30 | 每把武器独立 | 该武器弹药上限（MVP 默认 10） |
| `dual_ranged_damage` | int | — | 计算 | 双远程配置下单发伤害（基础伤害 × 0.9） |
| `source` | Node | — | 战斗系统 | 弹体伤害来源引用（指向玩家节点） |
| `overlapping_damage` | int | — | 计算 | 双近战重叠区域伤害 |

## Edge Cases

| 场景 | 预期行为 | 理由 |
|------|---------|------|
| 鼠标停在玩家位置上（aim_direction 为零向量） | 保持上一次的 aim_direction，不重置 | 防止鼠标静止时朝向丢失 |
| 近战扇形同时命中 5 个敌人 | 所有命中的敌人各自受到完整伤害 | 近战是范围攻击，不是单体 |
| 弹体飞行中玩家死亡 | 已发射的弹体继续飞行并正常命中 | 弹体是独立 Area2D 节点，不依赖玩家存活 |
| 弹药为 0 时按右键 | 该武器不生成弹体，无效果；另一把武器弹药不受影响 | 每把武器独立管理弹药 |
| 武器冷却中按攻击键 | 不触发攻击，忽略输入 | 冷却是硬性限制 |
| 同时按住左键和右键 | 只执行先按下的攻击类型，另一个忽略 | 互斥规则 |
| 远程射击时敌人刚进入射程 | 弹体已沿旧方向飞行，不会追踪新目标 | 弹道是直线，不追踪 |
| 近战攻击时玩家正在移动 | 近战扇形随玩家位置移动，但只在攻击瞬间检测命中 | 命中判定只在攻击帧 |
| 双近战只有一把冷却完成 | 只触发已冷却的那把，另一把忽略 | 独立冷却 |
| 弹体命中后敌人 HP ≤ 0 | 调用 take_damage()，敌人进入 DYING，由生命系统管理后续 | 战斗系统不负责死亡逻辑 |
| 弹体飞行超出射程 | 弹体自动销毁（queue_free），不造成伤害 | 射程限制 |
| 弹药回复期间射击 | 射击消耗弹药，**该武器**回复计时器重置；另一把武器不受影响 | 每把武器独立管理弹药和回复 |

## Tuning Knobs

| 参数 | 当前值 | 安全范围 | 调高效果 | 调低效果 |
|------|--------|----------|---------|---------|
| `melee_cooldown` | 0.5s | 0.2-1.5s | 近战节奏更慢，容错率更高 | 近战更快但可能过于强势 |
| `ranged_cooldown` | 0.3s | 0.1-1.0s | 远程节奏更慢，鼓励近战 | 远程更快，可能压制近战 |
| `melee_angle` | 120° | 60-180° | 近战覆盖范围更大 | 近战更精准，需要走位 |
| `melee_radius` | 60px | 30-100px | 近战攻击距离更远 | 近战需要更贴近敌人 |
| `bullet_speed` | 600 px/s | 300-1000 px/s | 弹体更快命中，手感更好 | 弹体更慢，需要预判 |
| `max_range` | 400px | 200-800px | 远程可以打更远的敌人 | 远程需要更靠近敌人 |
| `scatter_degrees` | 5° | 0-15° | 双远程更不准，风险更大 | 双远程更准，可能过于强势 |
| `max_ammo` | 10 | 5-30 | 弹药更充足，减少近战压力 | 弹药紧张，强制切换近战 |
| `ammo_regen_rate` | 1/s | 0.5-3/s | 弹药回复更快 | 弹药更紧张 |
| `dual_melee_overlap_bonus` | ×1.2 | 1.0-2.0 | 双近战重叠奖励更大 | 双近战收益降低 |
| `dual_ranged_cooldown_mult` | ×1.5 | 1.0-2.0 | 双远程射速变慢 | 双远程射速更快 |

> **注意**：以上 Tuning Knobs 中的武器相关参数（伤害、扇形角度/半径、弹道速度/散射、弹药上限）最终由**武器系统**提供。战斗系统只定义默认值和接口规范。弹药为**每把武器独立管理**，非全局共享。

| 近战命中 | 敌人闪烁红色 0.1s + 伤害数字飘字 | 命中音效（清脆切割） | 每个命中敌人独立触发 |

| 远程命中 | 敌人闪烁红色 0.1s + 伤害数字飘字 + 弹体命中粒子 | 命中音效（钝击） | 与近战命中音效区分 |
| 弹药耗尽 | 射击动作不触发，UI 弹药条闪烁红色 | 空弹音效（咔哒声） | 提示玩家需要回复 |
| 弹药回复 | 弹药条绿色填充动画 | 轻微充能音效 | 每秒回复时触发 |

## Acceptance Criteria

- [ ] 按住鼠标左键期间，近战武器按 0.5s 冷却循环触发攻击
- [ ] 近战攻击在玩家前方（鼠标方向）生成 120° 扇形碰撞区域，半径 80px
- [ ] 扇形内有 3 个敌人时，3 个敌人同时受到完整伤害
- [ ] 按住鼠标右键期间，远程武器按 0.3s 冷却循环发射弹体
- [ ] 远程弹体沿鼠标方向直线飞行，速度 600 px/s
- [ ] 远程弹体命中敌人后调用 `take_damage(15, "ranged", source)`，弹体销毁
- [ ] 远程弹体飞行超过 400px 后自动销毁
- [ ] 弹药为 0 时按右键不触发攻击，另一把武器不受影响
- [ ] 弹药 10 发全满时连续射击 10 次后无法继续射击，等待回复
- [ ] 该武器不射击 1 秒后回复 1 发弹药
- [ ] 武器冷却中按攻击键无效
- [ ] 同时按住左键和右键时，只执行先按下的攻击类型
- [ ] 人物朝向始终跟随鼠标方向，与 WASD 移动方向无关
- [ ] 鼠标静止不动时，人物保持最后一次朝向
- [ ] 双近战配置下，单击左键触发两把武器依次攻击（间隔 0.1s）
- [ ] 双近战扇形重叠区域内敌人受到两把武器伤害之和 × 1.2
- [ ] 双远程配置下，单击右键两把同时发射，每把消耗 1 发弹药，共享冷却（单把冷却 × 1.5）
- [ ] 双远程每把武器造成基础伤害的 90%（基础 15 → 每发 13）
- [ ] 双远程弹道带有 ±5° 随机散射，近距离基本命中，远距离可能空枪
- [ ] 玩家死亡（DYING 状态）后，战斗系统停止响应所有输入
- [ ] 已发射的弹体在玩家死亡后继续飞行并正常命中

## Open Questions

1. **近战攻击动画期间玩家能否移动** — 如果近战挥砍 0.2s 期间玩家完全不能移动，手感更重但操作受限；如果允许移动但速度降低，灵活性更高。MVP 阶段建议允许正常移动，playtest 后决定。

2. **武器系统的数据接口格式** — 战斗系统依赖武器系统提供武器数据，但具体接口（Resource 对象、字典、还是信号）待武器系统设计时确认。MVP 阶段在战斗系统内硬编码默认值。

3. **弹体是否需要穿透多个敌人** — 当前设计为命中第一个敌人后销毁。如果后续武器设计需要穿透弹（如狙击枪），需要在武器数据中增加 `pierce_count` 参数。

4. **近战攻击是否需要"攻击前摇"动画** — 当前近战攻击 0.2s 命中判定窗口从按下键立即开始。如果增加前摇（如 0.1s 准备 + 0.2s 命中），手感更重但需要预判。MVP 阶段无前摇，playtest 后评估。

5. **远程弹体是否需要重力/抛物线** — 当前设计为纯直线弹道。如果后续武器需要抛物线弹道，需要增加 `gravity` 参数。MVP 阶段不做。

6. **跨系统变更：移动系统朝向规则** — 本系统已覆盖移动系统的朝向规则（朝向 = aim_direction，而非移动方向）。需要在移动系统 GDD 中更新朝向规则，标注"朝向由战斗系统控制，移动系统不再管理朝向"。此项为跨系统同步事项，待后续批量处理。

# 玩家移动系统 (Player Movement System)

> **Status**: Approved
> **Author**: user + agents
> **Last Updated**: 2026-04-23
> **Implements Pillar**: 操作感即玩法、每一秒都在做选择

## Overview

玩家移动系统管理角色在 2D 竞技场中的位置变化。玩家通过 WASD/方向键在 8 个方向上以恒定速度移动，角色始终朝向移动方向。没有惯性、没有加速——按下键就立即以满速移动，松开键就立即停下。这种精确、即时的响应是操作感的基石。移动系统与碰撞检测、敌人追踪、近战攻击范围、闪避位移等所有战斗相关的系统直接交互。没有移动系统，战斗和闪避就无法运行。

*临时假设：输入系统提供标准化的 `move_vector: Vector2` 方向输入（-1 到 1），本系统将其转换为 8 方向并应用恒定速度。*

## Player Fantasy

你按下 W 键，角色像流水一样向前滑出。你在敌人群中穿梭——左、上、右、下——每一次转向都精确而流畅。你的角色始终面向你移动的方向，近战挥砍的弧线和远程射击的弹道都是从前方面朝方向发出。移动不是"走路"，而是**舞蹈**——你在刀尖上跳舞，用最小的位移躲开最多的攻击。当你在千钧一发之际从敌人攻击范围边缘滑过时，你会感到一种掌控一切的优雅。这个系统让你觉得自己是一个灵巧的战士，而不是一个笨拙的方块。

## Detailed Design

### Core Rules

1. **移动输入**
   - 从输入系统接收 `move_vector: Vector2`（范围 -1 到 1，x 为水平，y 为垂直）
   - 将原始输入转换为 8 方向：`direction = move_vector.normalized().snapped(Vector2(1,1).normalized())`
   - 如果 `move_vector` 长度 < 0.1（死区），视为无输入，不移动

2. **移动速度**
   - 基础速度：`300 px/s`
   - 对角线移动时自动归一化（`direction.normalized()`），确保斜向不比直向快
   - 实际位移 = `direction * base_speed * delta`（每帧计算）
   - 无加速/减速曲线——按下即满速，松开即停止

3. **朝向**
   - 角色始终朝向 `direction`（移动方向即面朝方向）
   - 无移动输入时保持上一次的朝向
   - 朝向用于近战攻击方向和远程射击方向

4. **碰撞**
   - 使用 Godot `CharacterBody2D` 的 `move_and_slide()` 处理移动和碰撞
   - 碰到竞技场墙壁时沿墙壁滑动（不卡住、不反弹）
   - 玩家碰撞体为 `CollisionShape2D`（圆形，半径 ~16px）
   - 玩家**不**与敌人碰撞体发生碰撞（可以穿过敌人）——只有墙壁和障碍物阻挡

5. **每帧执行流程**
   - 读取输入 → 转换为 8 方向 → 计算位移 → 应用碰撞 → 更新位置 → 更新朝向

### States and Transitions

| State | Entry Condition | Exit Condition | Behavior |
|-------|----------------|----------------|----------|
| `IDLE` | 无移动输入（move_vector 长度 < 0.1） | 收到移动输入 | 位置不变，保持上次朝向 |
| `MOVING` | 收到移动输入 | 移动输入停止 | 按 direction 以 base_speed 移动，朝向 = direction |

- 状态转换是即时的（无过渡动画）
- 闪避系统可以在 `MOVING` 期间覆盖移动方向和速度（由闪避系统控制）
- 死亡状态（由生命/伤害系统管理）下移动系统完全停止，不再响应输入

### Interactions with Other Systems

| 交互系统 | 方向 | 接口说明 |
|---------|------|---------|
| 输入系统 | 输入 → 本系统 | 提供 `move_vector: Vector2`；临时假设，待输入系统设计后验证 |
| 混合战斗系统 | 本系统 → 战斗 | 提供 `position` 和 `facing_direction`；战斗系统用于确定攻击方向和范围 |
| 闪避系统 | 闪避 ↔ 本系统 | 闪避期间覆盖移动方向和速度；闪避结束后恢复本系统控制 |
| 敌人 AI 系统 | 本系统 → 敌人AI | 提供 `position`；敌人 AI 用于追踪和包抄玩家位置 |
| 生命/伤害系统 | 本系统 ↔ 生命/伤害 | 死亡状态下移动系统停止响应；DYING 状态下角色留在原地不动 |
| 视觉反馈系统 | 本系统 → 视觉 | 提供 `position` 变化数据，用于移动轨迹粒子等特效 |

## Formulas

### 8 方向归一化

```
direction = move_vector.normalized().snapped(Vector2(1,1).normalized())
if direction.length() > 0:
    direction = direction.normalized()
```

### 位移计算

```
displacement = direction * base_speed * delta
new_position = current_position + displacement
```

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `move_vector` | Vector2 | (-1,-1) 到 (1,1) | 输入系统（临时假设） | 原始输入方向 |
| `direction` | Vector2 | 长度 0 或 1 | 计算 | 归一化后的 8 方向 |
| `base_speed` | float | 100-600 px/s | 调参旋钮 | 基础移动速度 |
| `delta` | float | ~0.0167 (60fps) | Godot 引擎 | 帧间隔时间 |

## Edge Cases

| 场景 | 预期行为 | 理由 |
|------|---------|------|
| 同时按下 W 和 D（对角线） | 归一化后沿东北方向移动，速度与单方向相同 | `direction.normalized()` 确保斜向不加速 |
| 输入值很小（如轻推摇杆 0.05） | 低于死区 0.1，视为无输入，不移动 | 防止摇杆漂移导致意外移动 |
| 贴着墙壁向墙壁方向移动 | 沿墙壁滑动，不会卡住 | `move_and_slide()` 自动处理 |
| 同时按下相反方向（如 W 和 S） | 输入抵消，`move_vector` 为零，不移动 | 自然抵消，符合直觉 |
| 闪避期间移动系统被覆盖 | 闪避系统控制移动，本系统不干扰 | 闪避优先级高于普通移动 |
| 玩家死亡（DYING/DEAD 状态） | 移动系统停止响应所有输入 | 死亡后不应再移动 |
| 一帧内多次位置请求 | 只应用最后一次，不叠加位移 | 每帧只计算一次位移 |
| 升级修改 `base_speed` 后 | 下一帧立即生效，无需重启游戏 | 数据驱动，支持实时调整 |

## Dependencies

| 系统 | 方向 | 性质 |
|------|------|------|
| 输入系统 | 本系统依赖输入 | 硬依赖：需要 `move_vector`（*临时假设接口，待验证*） |
| 混合战斗系统 | 战斗依赖本系统 | 硬依赖：读取 `position` 和 `facing_direction` |
| 闪避系统 | 闪避 ↔ 本系统 | 硬依赖：闪避期间覆盖移动方向和速度 |
| 敌人 AI 系统 | 敌人AI依赖本系统 | 硬依赖：读取 `position` 用于追踪 |
| 敌人生成系统 | 生成依赖本系统 | 硬依赖：读取 `position` 计算敌人生成位置 |
| 生命/伤害系统 | 本系统依赖生命/伤害 | 软依赖：查询死亡状态，DYING/DEAD 时停止移动 |
| 升级池/数据系统 | 升级 → 本系统 | 软依赖：`move_speed_bonus` 升级影响最终移动速度，最终速度 = base_speed × (1 + bonus) |
| 视觉反馈系统 | 视觉依赖本系统 | 软依赖：读取 `position` 变化用于移动特效 |

## Tuning Knobs

| 参数 | 当前值 | 安全范围 | 调高效果 | 调低效果 |
|------|--------|----------|---------|---------|
| `base_speed` | 300 px/s | 100-600 | 移动更快，闪避空间更大 | 移动更慢，需要更精确走位 |
| `input_deadzone` | 0.1 | 0.0-0.3 | 更宽容，防止误触 | 更灵敏，摇杆漂移可能影响 |
| `collision_radius` | 16 px | 8-32 | 碰撞体更大，更容易被墙壁挡住 | 碰撞体更小，更灵活 |

## Visual/Audio Requirements

| 事件 | 视觉反馈 | 音频反馈 | 说明 |
|------|---------|---------|------|
| 移动开始 | 角色播放行走动画（根据朝向切换方向） | 轻微脚步声循环 | 动画速度匹配移动速度 |
| 移动中 | 持续行走动画 | 持续脚步声，节奏固定 | 无加速所以节奏不变 |
| 移动停止 | 切回待机动画 | 脚步声停止 | 立即停止，无减速动画 |
| 贴墙滑动 | 角色轻微擦墙粒子 | 短促摩擦音效 | 视觉暗示碰撞发生 |
| 无输入待机 | 待机呼吸动画 | 无 | 保持上次朝向 |

## UI Requirements

移动系统本身不直接渲染 HUD 元素，但需要为 UI 系统暴露以下只读数据：

| 数据 | 类型 | 用途 |
|------|------|------|
| `position` | Vector2 | 小地图/迷你地图玩家标记位置 |
| `facing_direction` | Vector2 | 方向指示器箭头朝向 |
| `is_moving` | bool | 控制角色状态 UI 切换 |
| `base_speed` | float | 升级界面显示当前速度值 |

## Acceptance Criteria

- [ ] 只按 W 键时，角色以 300 px/s 向上移动，朝上
- [ ] 同时按 W+D 时，角色以 300 px/s 向东北移动（不是 424 px/s），朝东北
- [ ] 松开所有移动键后，角色立即停止（无惯性滑动）
- [ ] 松开所有移动键后，角色保持最后一次移动的朝向
- [ ] 角色碰到竞技场右墙壁时，如果同时按 W+D，沿墙壁向上滑动（不卡住）
- [ ] 输入值 < 0.1 时，角色不移动
- [ ] 同时按 W+S 时，角色不移动（输入抵消）
- [ ] 玩家死亡（进入 DYING 状态）后，移动键无效，角色停在原地
- [ ] 从静止到满速的过渡在 1 帧内完成（无加速动画）
- [ ] 移动系统的 `_physics_process` 每帧执行时间 < 1ms（60fps 下）

## Open Questions

1. **输入系统接口** — 当前假设输入系统提供 `move_vector: Vector2`。需要在输入系统 GDD 中确认接口名称、数据类型、是否已归一化。如果输入系统提供的是未归一化的原始值，本系统的归一化逻辑需要调整。

2. **角色美术方向数** — 角色行走动画是 4 方向还是 8 方向？如果是 4 方向，需要将 8 方向移动映射到最近的 4 方向动画（如东北 → 东）。

3. **移动速度升级叠加方式** — Roguelite 升级中的速度加成是累加（`base_speed += bonus`）还是乘法（`base_speed *= multiplier`）？这会影响 Tuning Knobs 的安全范围设计。

4. **_physics_process vs _process** — 移动逻辑应在 `_physics_process`（固定帧率，与物理同步）还是 `_process`（每帧渲染）中执行？默认推荐 `_physics_process` 以确保碰撞检测稳定，但需确认。

5. **游戏手柄摇杆死区曲线** — 线性死区还是环形死区？线性死区简单但手感生硬，环形死区更平滑但计算稍复杂。

# 战斗 HUD (Combat HUD)

> **Status**: Approved
> **Author**: user + agents
> **Last Updated**: 2026-04-27
> **Implements Pillar**: 操作感即玩法、每一秒都在做选择

## Overview

战斗 HUD 是玩家战斗时的信息仪表盘——在屏幕边缘持续显示生命值、分数、波次、弹药、闪避充能等关键数据，让玩家不必暂停就能掌握战局。本系统从各玩法系统读取数据并渲染 UI 元素，不产生任何游戏逻辑，不修改任何状态，它只做一件事：**让玩家随时知道自己处于什么状态**。

## Player Fantasy

你盯着屏幕中央的角色——头顶的血条还剩三分之一，该闪避了不是硬扛。左上角分数 23,400，Wave 4。角色身边弹药图标只剩 2 发，得省着用远程。闪避充能指示也在角色旁，亮了就能再滑一次。关键信息紧跟角色走，你的视线始终在战场中心，不用分心扫角落。

## Detailed Design

### 1. HUD 元素总览

| 元素 | 位置 | 空间类型 | 数据来源 |
|------|------|---------|---------|
| 生命条 | 角色头顶 | 世界空间 | 生命/伤害系统 |
| 弹药指示 | 角色身旁下方 | 世界空间 | 武器系统 |
| 闪避充能 | 角色身旁下方 | 世界空间 | 闪避系统 |
| 分数 | 屏幕左上角 | 屏幕空间 | 分数/统计系统 |
| 波次编号 | 屏幕左上角（分数下方） | 屏幕空间 | 波次系统 |
| 击杀数 | 屏幕左上角（波次下方） | 屏幕空间 | 分数/统计系统 |

### 2. 世界空间元素（跟随角色）

**生命条**：
- 位置：角色 sprite 正上方，距头顶约 20px
- 样式：水平条形，绿色→红色渐变（满血绿，低血红）
- 尺寸：宽 40px，高 4px，极简不遮挡视野
- 仅在 HP < 100% 时显示，满血时隐藏（减少视觉噪音）
- HP 变化时条形平滑过渡（0.2s Tween）

**弹药指示**：
- 位置：角色右下方
- 样式：小图标 + 数字（如 "🔫 3"）
- 弹药耗尽时数字变红并闪烁

**闪避充能**：
- 位置：角色左下方，与弹药对称
- 样式：小圆点指示器，充能完毕亮起，使用中暗淡
- 多充能时显示多个圆点（如 2 个圆点 = 2 次闪避可用）

### 3. 屏幕空间元素（固定位置）

**分数显示**：
- 位置：屏幕左上角，距边缘 20px
- 样式：白色数字，字体大小 20px
- 分数变化时数字短暂放大 + 弹回动画

**波次编号**：
- 位置：分数下方 8px
- 样式：`Wave {n}` 格式，字体大小 16px，灰白色
- 新波次开始时文字短暂放大淡入（1s），然后缩回正常

**击杀数**：
- 位置：波次下方 8px
- 样式：`Kills: {n}` 格式，字体大小 14px，灰白色

### 4. 波次提示弹窗

- 新波次开始时屏幕中央短暂显示 `Wave {n}` 大字，持续 2s 后淡出
- 升级窗口触发时显示 `Choose Upgrade` 提示（由局内升级系统自行渲染）

### 5. 数据更新方式

- **信号驱动**：监听 `score_changed`、`kill_count_changed`、`wave_started`、`wave_cleared` 信号即时更新
- **轮询更新**：HP、弹药、闪避充能每帧从对应系统读取（这些值变化频率高，信号过于频繁）
- **更新频率**：屏幕空间元素跟随信号，世界空间元素每帧更新

## Formulas

本系统无复杂公式，仅记录 UI 参数：

```
# 世界空间元素
HEALTH_BAR_WIDTH = 40          # 生命条宽度（px）
HEALTH_BAR_HEIGHT = 4          # 生命条高度（px）
HEALTH_BAR_OFFSET_Y = -20      # 生命条距角色头顶偏移（px，负值=上方）
HEALTH_BAR_TWEEN_DURATION = 0.2 # HP 变化过渡时长（秒）

# 屏幕空间元素
HUD_MARGIN = 20                # 屏幕边缘间距（px）
HUD_LINE_SPACING = 8           # 文字行间距（px）
SCORE_FONT_SIZE = 20           # 分数字号（px）
WAVE_FONT_SIZE = 16            # 波次字号（px）
KILLS_FONT_SIZE = 14           # 击杀数字号（px）

# 波次提示
WAVE_ANNOUNCE_DURATION = 2.0   # 波次提示显示时长（秒）
WAVE_ANNOUNCE_FONT_SIZE = 48   # 波次提示字号（px）
```

## Edge Cases

| 场景 | 预期行为 | 理由 |
|------|---------|------|
| 角色移动到屏幕边缘 | 世界空间元素跟随角色，可能部分超出屏幕 | MVP 不做边缘钳制，超出时可被遮挡 |
| HP 恢复到 100% | 生命条隐藏 | 减少视觉噪音，满血无需提示 |
| 弹药无限（近战模式） | 弹药指示器显示 ∞ 符号 | 区分无限弹药和有限弹药 |
| 闪避充能为 0 | 所有圆点暗淡，提示无法闪避 | 明确告知玩家当前状态 |
| 分数瞬间大幅变化（如连杀） | 数字直接跳到新值 + 放大弹回动画 | 不做逐帧递增，直接显示最终值 |
| 游戏暂停（升级窗口） | HUD 保持显示，数据冻结 | 暂停时 HUD 不消失 |
| 波次提示与升级界面重叠 | 波次提示先显示，升级界面打开时波次提示已淡出（时序不重叠） | 波次提示 2s 后消失，升级窗口在 wave_cleared 之后触发 |

## Dependencies

| 依赖系统 | 方向 | 依赖内容 | 状态 |
|---------|------|---------|------|
| 生命/伤害系统 | 上游 | 读取 HP / 监听伤害信号更新生命条 | GDD 已通过 ✅ |
| 武器系统 | 上游 | 读取当前弹药量 | GDD 已通过 ✅ |
| 闪避系统 | 上游 | 读取闪避充能状态 | GDD 已通过 ✅ |
| 分数/统计系统 | 上游 | 监听 `score_changed`/`kill_count_changed` 信号，读取分数/击杀数 | GDD 已通过 ✅ |
| 波次系统 | 上游 | 监听 `wave_started`/`wave_cleared` 信号，读取波次编号 | GDD 已通过 ✅ |
| 混合战斗系统 | 上游 | 读取当前武器类型（近战/远程） | GDD 已通过 ✅ |

**本系统无下游依赖**——HUD 是纯展示层，不向任何系统输出数据或信号。

## Tuning Knobs

| 旋钮 | 类型 | 默认值 | 安全范围 | 影响玩法 | 说明 |
|------|------|--------|---------|---------|------|
| `HEALTH_BAR_WIDTH` | int | 40 px | 20–80 | 信息密度 | 生命条宽度，太大遮挡视野，太小看不清 |
| `HEALTH_BAR_TWEEN_DURATION` | float | 0.2 s | 0.0–0.5 s | 反馈感 | HP 变化过渡速度，0=无过渡即时变化 |
| `WAVE_ANNOUNCE_DURATION` | float | 2.0 s | 1.0–4.0 s | 节奏感 | 波次提示停留时长，太长干扰战斗 |
| `HUD_MARGIN` | int | 20 px | 10–40 | 布局 | 屏幕边缘间距 |

**配置方式**：HUD 布局参数定义在 `assets/data/hud_config.tres`（Resource 文件）。

**备注**：字号和颜色属于视觉风格范畴，在 art bible 或 UI style guide 中统一管理，不作为代码旋钮。

## Visual/Audio Requirements

**视觉**：
- 极简几何风格，与游戏美术统一（方块、圆、线条）
- 生命条：绿→红渐变，圆角矩形
- 弹药/闪避：小图标 + 数字/圆点，半透明背景衬底提升可读性
- 分数/波次/击杀：等宽字体，白色/灰白色，无背景框
- 波次提示：大号白色字体，居中，无装饰
- 世界空间元素随角色缩放不缩放（固定像素尺寸，不随 zoom 变化）

**音频**：
- 无专属音效——HUD 是纯视觉层
- 波次提示的音效由波次系统委托音效系统播放，非 HUD 职责

## UI Requirements

本系统就是 UI 本身，Detailed Design 已详细定义所有元素。补充以下 UI 行为规范：

**显示/隐藏规则**：
- 战斗中：所有 HUD 元素常驻显示
- 升级窗口中：HUD 保持显示，数据冻结
- 死亡/结算：HUD 淡出（0.5s），让位给结算画面
- 主菜单：HUD 不显示

**无交互**：HUD 元素不可点击、不可聚焦，纯只读展示。玩家不通过 HUD 做任何操作。

**无重叠保证**：左上角信息区（分数/波次/击杀）垂直排列，间距固定，不会互相遮挡。世界空间元素间距由角色 sprite 大小保证。

## Open Questions

1. **生命条受伤闪烁**：角色受伤时是否需要生命条短暂白色闪烁？增强反馈感但增加实现量，MVP 可省略。
2. **低血量屏幕边缘特效**：HP 低于 25% 时屏幕边缘是否显示红色渐晕？对生存感知有帮助，但 MVP 可后置。
3. **世界空间元素边缘钳制**：角色靠近屏幕边缘时，头顶生命条可能超出视野。是否需要将世界空间元素钳制在屏幕内？MVP 不做，后续迭代处理。
